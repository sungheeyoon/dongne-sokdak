# 근접 그룹은 카카오 클러스터와 별도 개념이다 — 거리 고정, 줌으로 재계산하지 않음

지도 마커 밀도를 줄이기 위해 반경 30m 물리적 거리 기준의 근접 그룹(Proximity Group)을 도입한다. 카카오 클러스터(픽셀 격자 기반, 줌마다 소속 재계산)를 확장하는 대신 별도 개념으로 둔 이유는, 근접 그룹은 "같은 현장"이라는 물리적 사실을 표현하려는 것이라 줌 레벨과 무관하게 안정적으로 유지돼야 하기 때문이다. 그룹 소속은 제보 목록이 바뀔 때 한 번만 계산하고, 줌 레벨은 세 가지 표시 방식(카카오 클러스터 / 근접 그룹 마커 / 개별 마커) 중 어느 것을 그릴지 고르는 스위치 역할만 한다.

## Considered Options

- **카카오 클러스터의 grid 크기를 줄여서 흉내내기**: 픽셀 기반이라 줌마다 그룹 구성이 흔들리고, "이 제보들은 실제로 같은 현장"이라는 의미를 안정적으로 전달할 수 없어 기각.
- **체인(전이적) 방식으로 그룹 묶기** (A-B가 가깝고 B-C가 가까우면 A-C가 멀어도 한 그룹): 밀집 지역에서 그룹이 무한정 커질 수 있어 기각. 대신 시드 기반 그리디 방식(아직 그룹에 속하지 않은 제보를 시드로 삼아 그로부터 30m 이내인 것만 편입)으로 그룹 반경을 30m로 제한한다.
- **줌을 더 당기면 그룹을 강제로 낱개로 쪼개기**: 30m는 "같은 현장"으로 볼 수 있는 물리적 상한이므로 줌과 무관하게 그룹 소속을 유지하기로 했다. 대신 더 깊은 줌 구간(레벨 3 미만)에서는 그룹 데이터 자체를 무시하고 항상 개별 마커만 그린다 — 목적이 정확한 그룹 표현이 아니라 기본 화면의 마커 밀도를 낮추는 것이기 때문에, 충분히 확대한 뒤에는 그룹 표시가 오히려 불필요하다.

## Consequences

- 프로젝트에 없던 바텀시트 UI가 필요해져 `shared/ui/UiBottomSheet`를 범용 컴포넌트로 새로 만든다.
- 근접 그룹(2건 이상) 클릭은 카카오 클러스터 클릭의 pan/zoom 애니메이션(`getFitLevel` 기반 확대)을 재사용하지 않는다 — 그룹은 줌인해도 갈라지지 않으므로 이동 애니메이션 없이 즉시 바텀시트를 연다. 그룹 크기가 1인 경우만 기존 개별 마커 클릭 동작(레벨 3 포커스 + 상세 모달)을 그대로 따른다.
